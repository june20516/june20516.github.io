문제 풀기 전략

  1. 문제를 잘읽는다
  2. 답을 구한다 (가능한 간단히, 가능한 쉬운답)
  3. 복잡도 분석
  4. input/제한시간이 통과 되는지 확인(3.을 통해)
  5. 시간이 초과되면 복잡도를 낮추거나, 코드를 개선해야한다. (5n^2 -> 4n^2 같은 것은 의미 없다. 복잡도를 낮춰야 한다. n^2 -> n )
  6. 복잡도 낮추려면, 지금 방법과는 완전히 다른 방법을 생각해야함. 자료구조!!
